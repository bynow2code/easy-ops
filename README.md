# EasyOps 脚本管家

> 一个帮你**集中管理、随手执行 Shell 脚本**的桌面小工具。把零散的运维/开发脚本收进一个窗口，分类、编辑、一键运行、实时看日志，不用再开着终端敲命令。

---

## 这是什么，解决什么问题

日常开发中我们手里总有一堆零散脚本：启动后端、拉起前端、数据库备份、清理缓存、部署发布、定时巡检……它们散落在各个文件夹里，时间一长就**记不住放哪、想不起参数、复制粘贴还容易敲错**。

EasyOps 把这些脚本统一管起来：

- **不再翻文件夹**：所有脚本都在一个窗口里，按分组排列。
- **不用开终端**：点一下就能跑，输出实时滚出来，跟看终端一模一样。
- **不怕忘**：脚本内容、参数都保存在应用里，下次直接用。
- **批量省事**：多选几个脚本，一键并发执行，适合"启动一整套环境"这类场景。

简单说，它是一个**给脚本用的"启动器 + 管理器"**，让重复性的命令操作变成点点鼠标的事。

## 核心功能

| 功能 | 说明 |
|------|------|
| 📁 **脚本管理** | 新增、编辑、删除 Shell 脚本，内容随手保存 |
| 🗂️ **分组管理** | 支持 Backend / Frontend 等分组，拖拽即可切换所属分组 |
| ↕️ **拖拽排序** | 拖动调整脚本顺序，支持跨分组拖拽 |
| ⚡ **实时流式执行** | 基于 SSE 技术，脚本输出像终端一样实时滚动展示 |
| 🔁 **批量执行** | 多选脚本后一键批量运行，多个脚本可并发执行 |
| 🎨 **语法高亮编辑器** | 内置 CodeMirror 6，写 Shell 脚本有高亮、更清爽 |
| 💻 **跨平台** | 自动检测 Bash / Git Bash / WSL / cmd，Windows、macOS、Linux 都能跑 |
| 🔔 **系统通知** | 脚本跑完自动弹出系统原生通知，不用一直盯着窗口 |
| 🧹 **一键关闭** | Execution Outputs 面板支持一键关闭所有输出 |
| ℹ️ **系统信息** | App Info 里查看当前 Shell 类型、路径、版本等 |
| 🔄 **自动更新** | 启动时可检查新版本，更新日志按 GitHub Release 风格展示 |

## 怎么用

### 1. 安装

- **Windows**：下载安装包（`.exe`，NSIS）双击安装；或下载 ZIP 便携版解压即用。
- **macOS**：下载 `.dmg` 安装（同时提供 Intel 与 Apple Silicon 版本）。
- **Linux**：（按需提供对应包体）

> 安装完成后，应用图标会出现在开始菜单 / 应用程序 / 任务栏中。

### 2. 新建并编写脚本

1. 点击 **新建脚本**（或类似入口）。
2. 给脚本起个名字，例如「启动后端服务」。
3. 选择分组（Backend / Frontend）。
4. 在编辑器里写下你的 Shell 命令，例如：

   ```bash
   cd /path/to/project
   npm run dev
   ```

5. 保存。脚本就出现在左侧列表里了。

### 3. 运行脚本

- 在脚本上点击 **运行**（▶），下方 **Execution Outputs** 面板会实时显示执行日志，和终端体验一致。
- 脚本执行结束后，系统会弹出通知提醒你。

### 4. 批量执行

- 勾选多个脚本，点击 **批量执行**，它们会并发跑起来，各自有独立的输出面板。

### 5. 调整顺序与分组

- 直接**拖拽**脚本条目，可以调整执行顺序，也可以拖到另一个分组里。

### 6. 查看信息与更新

- 通过菜单打开 **App Info**，可看到当前 Shell 类型、可执行文件路径、版本号等。
- 通过菜单 **检查更新**，可手动触发版本检查并查看更新日志。

## 常见问题

**Q：脚本执行时会弹出终端窗口吗？**
不会。应用在 Windows 上以隐藏窗口方式运行 Shell，执行过程只显示在应用内的输出面板中，不会跳出黑框或 Windows Terminal。

**Q：我的脚本用的是 cmd 还是 bash？**
应用会自动检测当前环境：Windows 优先 Git Bash / WSL，回退到 cmd；macOS / Linux 使用 bash。可在 App Info 中查看当前实际使用的 Shell。

**Q：数据存在哪？**
脚本内容保存在应用数据目录下（开发模式在 `server/scripts.json`），卸载应用前请按需备份。
